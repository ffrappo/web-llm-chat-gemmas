### Observations

- **GPU inference is fast**: Short-answer tasks complete in 0.1–1.6s. Longer structured outputs (JSON, CSV) take 3–14s due to higher `max_new_tokens` generation.
- **Arithmetic failure**: Model answered "3" instead of "2" for the apple word problem. This suggests the 2B parameter model still struggles with multi-step arithmetic reasoning.
- **Instruction hierarchy failure**: Model followed the first instruction (capital of Japan) and appended "NO" instead of overriding with just "NO". This is a known challenge for smaller models.

## How to Reproduce

### 1. Provision an instance

Using the Thunder Compute API:

```bash
export TNR_API_TOKEN="your-token"
curl -s -X POST \
  -H "Authorization: Bearer $TNR_API_TOKEN" \
  -H "Content-Type: application/json" \
  https://api.thundercompute.com:8443/v1/instances/create \
  -d '{
    "cpu_cores": 8,
    "disk_size_gb": 100,
    "gpu_type": "a100xl",
    "mode": "prototyping",
    "num_gpus": 1,
    "template": "base"
  }'
```

### 2. Set up the environment

```bash
ssh -p 30605 -i ~/.thunder/gemma-bench.key ubuntu@38.80.122.137
# Then on the remote:
curl -fsSL https://raw.githubusercontent.com/fornacestudio/gemmas/main/scripts/remote-setup.sh | bash
```

### 3. Run the GPU benchmark

```bash
ssh -p 30605 -i ~/.thunder/gemma-bench.key ubuntu@38.80.122.137 \
  "cd ~/gemmas && python3 scripts/bench-gpu.py --suite"
```

Or use the local wrapper:

```bash
export GEMMAS_REMOTE="ubuntu@38.80.122.137"
export GEMMAS_REMOTE_DIR="/home/ubuntu/gemmas"
./scripts/remote-run.sh --gpu --suite --output results.json
```
